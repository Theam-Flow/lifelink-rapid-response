## Development Principles

### What Guides Our Decisions

1. **Life-Saving First**
   - Features that save lives get priority
   - Emergency fixes interrupt everything
   - User safety over user comfort

2. **Open & Free**
   - No features locked behind paywalls
   - No proprietary dependencies
   - Always open source

3. **Simple Over Complex**
   - Easy to use in crisis
   - Works on old devices
   - Low-bandwidth friendly

4. **Community-Driven**
   - Listen to actual users
   - Prioritize common needs
   - Support diverse use cases

5. **Sustainable**
   - Build for long-term
   - Avoid technical debt
   - Maintainable codebase

## Release Schedule

### Versioning

We use Semantic Versioning: `MAJOR.MINOR.PATCH`

- **MAJOR**: Breaking changes
- **MINOR**: New features (backward compatible)
- **PATCH**: Bug fixes

### Release Frequency

- **Patch releases**: Weekly (bug fixes)
- **Minor releases**: Monthly (new features)
- **Major releases**: Quarterly (big changes)

### Support Policy

- **Current version**: Full support
- **Previous version**: Security patches only
- **Older versions**: Community support only
